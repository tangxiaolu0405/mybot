# CLAW Project Agent

## 项目概述

CLAW (Cognitive Learning and Adaptive Workspace) 是一个智能工作空间项目，旨在构建具有学习能力和自适应机制的AI代理系统。

## 核心设计理念

### 1. 好品味 (Good Taste)
- 消除边界情况，使特殊情况消失成为正常情况
- 优先使用简洁的无条件分支而非复杂的条件判断
- 追求代码的直观性和优雅性

### 2. 不破坏用户空间 (Never Break Userspace)
- 任何导致现有功能失效的改动都是bug
- 向后兼容性是神圣不可侵犯的
- 系统的职责是服务用户，而不是教育用户

### 3. 实用主义 (Pragmatism)
- 解决实际问题，而非假想的威胁
- 拒绝理论完美但实际复杂的方案
- 代码为现实服务，不为论文服务

### 4. 简洁执念 (Simplicity Obsession)
- 函数必须短小精悍，只做一件事并做好
- 避免超过3层缩进的代码结构
- 清晰的代码优于聪明的代码

## 项目规范

### 代码风格
- 遵循语言特定的代码规范
- 保持一致的命名约定
- 优先使用现有的库和工具
- 避免过度工程化

### 架构原则
- 模块化设计，职责分离
- 最小化依赖
- 优先组合而非继承
- 接口优于实现

### 质量标准
- 代码必须通过lint检查
- 类型检查必须通过
- 测试覆盖率保持在高水平
- 文档与代码同步更新

## 项目结构

```
claw/
├── agent/              # 项目agent配置
├── brain/              # 核心认知系统
│   ├── core.md        # 核心认知架构
│   └── memory/        # 记忆存储
│       ├── long-term/ # 长期记忆
│       └── short-term/ # 短期记忆

│   └── *.md           # 各个技能的SKILL.md文件
├── scripts/           # 工具脚本

```

## 开发流程

1. **需求分析** - 明确问题边界和用户需求
2. **方案设计** - 基于实用主义选择最优方案
3. **实现开发** - 遵循简洁原则编写代码
4. **质量验证** - 运行测试和lint检查
5. **文档更新** - 保持文档与代码同步

## 技术栈

- **核心语言**: Python / JavaScript (根据具体需求选择)
- **AI框架**: Claude API集成
- **存储**: Markdown文件系统
- **版本控制**: Git

## 质量保证

- 每次提交前运行lint和typecheck
- 保持测试覆盖率 > 80%
- 代码审查必须通过
- 文档必须完整准确

## 扩展性

- 插件化技能系统
- 模块化记忆架构
- 可扩展的agent配置
- 灵活的集成接口
