# CLAW Skills Specification

**重要说明**: 本规范用于指导AI agent创建和管理CLAW项目中的自定义技能。所有技能描述应保持简洁，避免冗长的示例、模板和详细说明，以减少context消耗。

## 核心原则

1. **简洁性**: 技能描述简洁明了，指令清晰易懂，避免过度复杂
2. **实用性**: 解决实际问题，提供明确价值，易于使用和理解
3. **可扩展性**: 预留扩展点，支持技能组合，保持向后兼容
4. **可维护性**: 文档完整准确，代码结构清晰，便于更新维护

## 技能格式规范

### YAML Frontmatter（必需）

```yaml
---
name: skill-name
description: 技能的详细描述
version: 1.0.0
author: 作者名
tags: [tag1, tag2]
dependencies: []
---
```

**必需字段**：
- `name`: 技能唯一标识符，小写，连字符分隔
- `description`: 技能的详细描述
- `version`: 版本号（语义化版本）

**可选字段**：
- `author`: 作者名称
- `tags`: 技能标签数组
- `dependencies`: 依赖的其他技能数组

### Markdown 内容结构

必须包含以下部分：

```markdown
# 技能名称

## 功能描述
[描述技能的功能、解决的问题、主要特性和适用范围]

## 使用场景
- 场景1: 描述
- 场景2: 描述

## 指令说明
### 基础指令
- 指令1: 说明
- 指令2: 说明

### 高级指令
- 指令3: 说明

## 最佳实践
- 实践1
- 实践2

## 注意事项
- 注意1
- 注意2
```

## 命名约定

### 技能名称规则
- 使用小写字母
- 单词间用连字符（-）分隔
- 描述性强且简洁
- 避免特殊字符和空格

**正确示例**: `code-generator`, `document-parser`, `data-analyzer`
**错误示例**: `CodeGenerator`, `code_generator`, `code generator`

### 目录名称
目录名称与技能名称保持一致

### 文件名称
- 主文件：`SKILL.md`（大写）
- 示例文件：`example-*.md`
- 资源文件：根据类型命名（如`config.json`, `data.csv`）
- 测试文件：`test-*.md`或`test-*.py`

## 目录结构规范

### 标准技能目录

```
skill-name/
├── SKILL.md              # 技能主文件（必需）
├── examples/             # 使用示例（可选）
│   ├── example-1.md
│   └── example-2.md
├── resources/            # 资源文件（可选）
│   ├── config.json
│   └── data.csv
└── tests/               # 测试文件（可选）
    ├── test-basic.md
    └── test-advanced.md
```

## 内容规范

### 功能描述
- 清晰说明技能的功能
- 说明解决的问题
- 列出主要特性
- 说明适用范围

### 使用场景
- 列出典型使用场景
- 每个场景独立成项
- 提供场景描述
- 说明预期效果

### 指令说明
- 描述指令功能
- 列出参数说明
- 提供使用示例
- 说明返回值

### 最佳实践
- 列出推荐做法
- 说明原因和效果
- 提供实施建议
- 避免常见错误

### 注意事项
- 列出潜在问题
- 提供解决方案
- 说明预防措施
- 标记重要警告

## 版本控制

### 版本号格式
遵循语义化版本：`MAJOR.MINOR.PATCH`
- `MAJOR`: 不兼容的API修改
- `MINOR`: 向下兼容的功能性新增
- `PATCH`: 向下兼容的问题修正

### 版本更新规则
- 破坏性变更：更新`MAJOR`版本
- 功能新增：更新`MINOR`版本
- 问题修复：更新`PATCH`版本

## 依赖管理

### 依赖声明
在YAML frontmatter中声明依赖：

```yaml
dependencies:
  - skill-a
  - skill-b
```

### 依赖规则
- 只声明必需的依赖
- 避免循环依赖
- 明确依赖版本
- 提供替代方案

## 质量标准

### 格式检查清单
- YAML frontmatter完整
- 必需字段齐全
- Markdown格式正确
- 无语法错误

### 内容检查清单
- 描述清晰准确
- 指令明确
- 文档完整

### 功能检查清单
- 按预期工作
- 错误处理完善
- 性能可接受
- 无安全漏洞

## 安全规范

### 输入验证
- 验证输入类型
- 检查输入范围
- 过滤危险内容
- 处理异常输入

### 输出控制
- 限制输出长度
- 过滤敏感信息
- 控制输出格式
- 验证输出内容

### 权限控制
- 最小权限原则
- 访问控制检查
- 操作审计日志
- 异常行为监控

## 性能规范

### 响应时间
- 基础操作 < 1秒
- 复杂操作 < 10秒
- 批量操作 < 60秒

### 资源使用
- 内存使用合理
- CPU占用可控
- 网络请求优化
- 缓存有效利用

## 兼容性规范

### 向后兼容
- 不破坏现有功能
- 保持API稳定
- 支持旧版本
- 提供迁移指南

### 跨平台
- 支持主流平台
- 处理平台差异
- 统一接口设计
- 平台特定优化
